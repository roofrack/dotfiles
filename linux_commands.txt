virtualenv commands...
----------------------

set project dir when activating virtenv
activate the environment, cd into dir and then type: setvirtualenvproject

to start a env using python3...
virtualenv -p /usr/bin/python3 env_name


can also use python3 built in venv...
this is the more correct way rather than virtualenvwrapper. Not sure
why but when using python3 should use this. Cd into your project and
type the following to start a new enviroment (usually called venv).

pyvenv /path/to/where_you_want/<venv>  
source <venv>/bin/activate  (this activates the env)
deactivate (to deactivate the env)
